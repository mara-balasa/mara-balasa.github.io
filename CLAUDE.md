# Claude Code — Project Rules & Guardrails

## Scope & File Safety

- Only read, create, or modify files within this project folder. Never touch files outside of it.
- Never delete files without explicitly asking first and waiting for confirmation.
- Never overwrite existing files without showing what will change first.
- Always create a backup note if making large structural changes.

## Code Quality

- Always write clean, readable, well-commented code.
- Keep components modular — one purpose per file.
- Reuse existing styles and components before creating new ones.
- Always make the site fully responsive for mobile and desktop.

## Dependencies

- Always ask before installing a new package or dependency.
- Prefer lightweight solutions — avoid large libraries if the same result can be achieved simply.
- Never install deprecated or insecure packages.

## Communication

- Before making large changes, summarise what you are about to do and wait for approval.
- If something is unclear, ask rather than making assumptions.
- After completing a task, give a short summary of what was changed.
- If an error cannot be fixed in 2 attempts, stop and explain the problem.

## Git & Version Control

- Do not run any git commands unless explicitly asked.
- Never push to any remote repository.

## Allowing
- Always allow to edit the contact.html cv.html index.html teaching.html and research.html from the Website folder we work in, without asking for explicit approval from me when I run a line of code

## Updating
- Always update the stats with the number of publications, working papers, journal articles
